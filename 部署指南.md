# JW-DSAR 部署指南（Linux + Windows；JW-Flare 仅 Linux）

本文档将项目拆分为两块独立部署：

- **A. JW-DSAR 主体**：可迁移到任意 Linux/Windows 主机
- **B. JW-Flare 服务**：只在 Linux 服务器端运行（可同机或独立服务器）

## 1. 架构与解耦

- JW-DSAR 通过 `JWFLARE_API_BASE` 调 JW-Flare 的 OpenAI 兼容接口
- 传图支持三种模式（`JWFLARE_TRANSPORT`）：
  - `upload`：跨机推荐（上传图片内容）
  - `path`：兼容模式（传路径字符串，需服务端可读该路径）
  - `auto`：先 `upload`，失败后可回退 `path`

## 2. 安装与基础配置

```bash
conda create -n jw-dsar python=3.10 -y
conda activate jw-dsar
pip install -U pip
pip install -r requirements_scheduled.txt
# 可选：JW-Flare链路
# pip install -r requirements_jwflare.txt
cp .env.example .env
```

已验证可用的 Gradio 版本：**4.44.1**。

## 2.1 通过 GitHub 复现（含数据 + Git LFS）

本项目可能包含 `data/` 下的压缩包（使用 Git LFS 存储）。在新机器上复现建议：

```bash
git clone <your_repo_url>
cd JW-DSAR
git lfs install
git lfs pull
cp .env.example .env
```

`.env` 推荐起步配置：

```env
DASHSCOPE_API_KEY_FILE=./secrets/dashscope_api_key.txt
DASHSCOPE_VL_MODEL=qwen3.6-plus

JWFLARE_ENABLED=1
JWFLARE_DATA_ROOT=./data
JWFLARE_API_BASE=http://127.0.0.1:2222/v1
JWFLARE_MODEL=qwen2-vl-7b-instruct
JWFLARE_MAX_REGIONS=3
JWFLARE_TRANSPORT=upload
JWFLARE_UPLOAD_FORMAT=base64
JWFLARE_ALLOW_PATH_FALLBACK=0
```

## 3. 一行启动（JW-DSAR 主体）

### Linux

长期运行（推荐 systemd）：

```bash
bash scripts/install_systemd_units.sh
```

```bash
python app_scheduled.py
```

单次生成（定时任务/批处理）：

```bash
python app_scheduled.py --generate-once
```

### Windows

```bat
scripts\start_jwdsar_windows.bat
```

单次生成：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/run_daily_report_windows.ps1
```

长期运行（计划任务，一键安装）：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/install_windows_tasks.ps1
```

## 4. 一行启动（JW-Flare 服务，仅 Linux）

### Linux

```bash
  # 默认会尝试用 conda run -n qwen 执行（可用 JWFLARE_CONDA_ENV 覆盖）
  # 下面给出一个你当前环境的 ckpt_dir 示例（按实际替换）：
  # /data/shaomf/JW-Flare/output/qwen2-vl-7b-instruct/v43-20241025-184811/checkpoint-180-merged
JWFLARE_CKPT_DIR=/path/to/checkpoint bash scripts/start_jwflare.sh
```

可选环境变量：

- `JWFLARE_PORT`（默认 `2222`）
- `JWFLARE_SWIFT_BIN`（默认 `swift`）
- `JWFLARE_LOG_DIR` / `JWFLARE_LOG_FILE`
- `CUDA_VISIBLE_DEVICES`
- `SIZE_FACTOR` / `MAX_PIXELS`

## 5. 跨机部署要求（重点）

当 JW-DSAR 与 JW-Flare 不在同一台机器时：

1. `JWFLARE_API_BASE` 必须可网络访问（放通端口）
2. 推荐 `JWFLARE_TRANSPORT=upload`
3. 若使用 `path` 模式，必须满足共享存储 + 路径一致
4. `JWFLARE_MODEL` 必须和服务端实际模型一致
5. 远程延迟较高时提高 `JWFLARE_HTTP_TIMEOUT`

## 6. systemd 定时运行（Linux）

仓库自带模板：

- `deploy/systemd/jwdsar-web.service`
- `deploy/systemd/jwdsar-generate.service`
- `deploy/systemd/jwdsar-generate.timer`

一键安装：

```bash
bash scripts/install_systemd_units.sh
```

安装脚本会自动按当前仓库路径渲染 unit，并支持变量覆盖：

- `JWDSAR_PYTHON_BIN`
- `JWDSAR_SERVICE_USER`
- `JWDSAR_SERVICE_GROUP`
- `JWDSAR_SECRETS_DIR`

常用运维命令：

```bash
systemctl status jwdsar-web.service
systemctl list-timers --all | rg jwdsar-generate
journalctl -u jwdsar-web -f
journalctl -u jwdsar-generate --since today
```

## 7. 数据目录与输出目录

- 日级缓存：`data/YYYY-MM-DD/`
- 年归档：`data/SRS/`、`data/events/`
- 报告目录：默认 `reports/`
  - 可通过 `JWDSAR_REPORTS_DIR` 配置
  - 或运行时使用 `--output-dir`

## 8. 回归测试建议

```bash
python app_scheduled.py --generate-once --date 2026-04-02 --output-dir test/reports
```

检查项：

- `test/reports/` 生成 `report_YYYY-MM-DD.html`
- 同日期二次生成优先复用 `data/YYYY-MM-DD/` 快照
- 开启 JW-Flare 时 section 3 后出现 JW-Flare 结果片段

## 9. 常见问题

- **跨机无结果**：优先切到 `JWFLARE_TRANSPORT=upload`
- **PDF 失败**：缺少系统依赖（Pango/Cairo），先保证 HTML 可生成
- **JW-Flare 无输出**：检查 `JWFLARE_API_BASE`、`JWFLARE_MODEL`、端口连通性
- **历史数据未命中**：检查 `data/SRS`、`data/events`、`data/YYYY-MM-DD` 布局
